# 공격 의도 표시 타이밍 수정 ✅

## 🔄 타이밍 흐름

### 이전 (잘못됨)
```
몬스터 의도 선택 → 미리보기 표시
몬스터 공격 → 미리보기 숨김
```
❌ 플레이어가 몬스터 공격을 예측할 수 없음

---

### 수정 후 (올바름)
```
1️⃣ 플레이어 턴 시작
   → 몬스터 공격 의도 미리보기 표시 ✅
   (플레이어가 다음 공격 예측 가능)

2️⃣ 플레이어 행동
   → 공격 의도 계속 표시됨

3️⃣ End Turn 클릭
   → 공격 의도 미리보기 숨김

4️⃣ 몬스터 턴
   → 공격 실행
   → 다음 의도 선택
   
5️⃣ 다시 플레이어 턴 시작
   → 새 공격 의도 표시
```

---

## ✅ 수정 사항

### 1. TurnManager.StartPlayerTurn()
- ✅ `ShowMonsterIntents()` 추가
- 플레이어 턴 내내 공격 의도 표시

### 2. TurnManager.EndPlayerTurn()
- ✅ `HideMonsterIntents()` 추가
- 몬스터 턴 전에 미리보기 숨김

### 3. Monster.ExecuteIntent()
- ✅ `HideAttackPreview()` 제거 (중복)
- TurnManager가 관리
